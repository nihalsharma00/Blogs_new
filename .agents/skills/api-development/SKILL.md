# API Development Skill

Use this skill when adding or changing API endpoints.

## Process

1. Define route and HTTP method.
2. Define auth and permission requirements.
3. Define request and response schemas.
4. Add validation.
5. Add route.
6. Add controller.
7. Add service.
8. Add database operation.
9. Add error mapping.
10. Add tests.
11. Add frontend integration where required.

## Mandatory Rules

- Use `/api/v1`.
- Validate all external input.
- Keep controllers thin.
- Enforce permissions on the server.
- Never expose internal errors or secrets.
